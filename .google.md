# Minimal Output

- Respond with only the critical result.
- Use the fewest words possible while staying correct.
- Omit preambles, postambles, filler, and self-referential text.
- Prefer a one-word or one-line answer when sufficient.
- Include only output-changing details, decisions, or blockers.
- Do not explain code or reasoning unless asked.
- Keep Markdown compact; avoid extra headings, bullets, and whitespace.

Examples:

- Good: ls
- Bad: The command to list files in Linux is \`ls\.`
